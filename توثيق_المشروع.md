# توثيق مشروع منصة وتطبيق Million Platform / Nexus EDU

هذا الملف يوثق بوضوح ما تم إنجازه في المشروع والتقنيات والأدوات التي تم استخدامها في بناء المنصة المتكاملة (الويب، الواجهة الخلفية، والتطبيق). تم جمع هذه المعلومات بناءً على تحليل بنية الكود وملفات التكوين الخاصة بالمشروع.

## 1. بنية المشروع (Architecture)
تم الاعتماد على بنية **Monorepo** باستخدام تقنية **Turborepo** لإدارة أجزاء المشروع المختلفة (الويب، التطبيق، الواجهة الخلفية، الحزم المشتركة) في مستودع كود واحد. هذه الاستراتيجية تُسهل مشاركة الأكواد والإعدادات (مثل ESLint و TypeScript) بين مختلف أجزاء النظام.

## 2. الواجهة الخلفية (Backend - API) `apps/api`
تم بناء الواجهة الخلفية لكي تكون قوية وقابلة للتوسع (Scalable) لدعم آلاف المستخدمين، بالاعتماد على أحدث التقنيات:
- **إطار العمل:** NestJS (Node.js) لبنية قوية تعتمد على TypeScript.
- **قواعد البيانات:** PostgreSQL مع استخدام Prisma ORM لإدارة البيانات والعلاقات بطريقة آمنة وسريعة.
- **التخزين المؤقت وقوائم الانتظار:** استخدام Redis و BullMQ لمعالجة المهام في الخلفية (Background Jobs) مثل إرسال الإشعارات والبريد الإلكتروني، وعمليات النظام الأخرى.
- **الاتصال اللحظي (Real-time):** الاعتماد على Socket.io لتمكين الميزات التفاعلية مثل الدردشة، الفصول المباشرة، والتحديثات اللحظية للإشعارات والمنافسات.
- **إدارة الملفات الدائمة:** استخدام Multer لإدارة تخزين ورفع الملفات.
- **المصادقة والأمان:** استخدام JWT (JSON Web Tokens) و Passport.js للمصادقة، بالإضافة إلى تشفير كلمات المرور باستخدام Bcrypt وتأمين واجهات الـ API عبر Helmet وتقييد عدد الطلبات للحد من الهجمات (Rate limiting).
- **الذكاء الاصطناعي (AI):** دمج مكتبة `openai` لتقديم حلول الذكاء الاصطناعي (مساعد شخصي للطلاب ومراجعة الإجابات).
- **المدفوعات:** تكامل مع Stripe لمعالجة عمليات الدفع والاشتراكات.

## 3. منصة الويب (Web - Frontend) `apps/web`
الواجهة الأمامية للمنصة (لوحة تحكم المعلمين والمديرين، وواجهة الطلاب على الويب) تم تطويرها لتكون سريعة وعالية الأداء وتوفر تجربة مستخدم (UX) ممتازة:
- **إطار العمل الرئيسي:** Next.js (الإصدار الحديث 16) باستخدام React 19.
- **تنسيق الواجهات (Styling):** الاعتماد على Tailwind CSS لبناء تصميمات متجاوبة وسريعة، مع استخدام واجهات جاهزة وعالية الجودة من Shadcn/ui و Radix UI.
- **إدارة الحالة وجلب البيانات:** استخدام Zustand لإدارة الحالة داخل التطبيق بسهولة، و React Query و React Table لإدارة وجلب وتخزين البيانات من الـ API بشكل فعال (Caching & Data Grids).
- **التواصل المباشر (المكالمات والبث):** استخدام مكتبة Jitsi (Jitsi React SDK) لإنشاء وتضمين الفصول الافتراضية المباشرة داخل المنصة بشكل احترافي وبزمن انتقال منخفض (Low-latency).
- **المصادقة:** تفعيل NextAuth.js للمصادقة وتأمين الجلسات في بيئة الويب بشكل آمن.
- **الرسوم البيانية:** مكتبة Recharts لعرض الإحصائيات لمتخذي القرار.

## 4. تطبيق الهاتف الموبايل (Flutter App - Nexus EDU) `apps/flutter_app`
تم بناء تطبيق للهواتف الذكية (ويدعم منصات أخرى) مخصص للطلاب باسم **"Nexus EDU"**:
- **إطار العمل:** تقنية Flutter (بإصدار حديث للغة Dart) لعمل تطبيق يعمل على منصات متعددة (Android / iOS) بكود واحد.
- **إدارة الحالة:** Riverpod كواحد من أحدث وأفضل حلول إدارة الحالة في فلاتر (لضمان أداء عالٍ وكود نظيف).
- **توجيه الصفحات والتنقل:** GoRouter لبناء نظام توجيه حديث مبني على الروابط والـ Deep Linking.
- **ربط الـ API:** حزمة Dio القوية للتعامل مع الشبكة وإرسال الطلبات للواجهة الخلفية.
- **التخزين وسرعة الاستجابة:** استخدام SharedPreferences لحفظ التفضيلات محلياً، و Cached Network Image لتحميل الصور وحفظها في الـ Cache لتحسين أداء وعرض التطبيق وعدم استهلاك الإنترنت.
- **التصميم والرسوم البيانية:** استخدام Fl_chart لعمل رسوم بيانية تفاعلية لعرض إحصائيات تقييم الطالب ومساره التعليمي بشكل مميز.

## 5. البنية التحتية والاستضافة (Deployment & Infrastructure)
- **الاستضافة والنشر:** 
  - Vercel لاستضافة واجهة الويب (Next.js) لسرعتها الهائلة على مستوى العالم.
  - إعدادات جاهزة (مثل `Procfile` وغيرها) لاستضافة الواجهة الخلفية وقواعد البيانات على منصات مثل Railway.
- **الحاويات (Containers):** استخدام Docker (تجهيز ملفات `docker-compose.yml` و `docker-compose.prod.yml`) لتشغيل بيئة التطوير محليًا بسرعة وتسهيل عملية النشر وإدارة قواعد البيانات.
- **أدوات مساعدة للتطوير:** ESLint و Prettier لتوحيد أسلوب كتابة الكود عبر الفريق وتقليل الأخطاء البرمجية.

## 🎯 ملخص الميزات والمخرجات التي تم إنجازها في النظام:
1. **نظام شامل لإدارة التعلم:** منصة مركزية متكاملة لإضافة المواد الدراسية، إدارة الحصص، والمدرسين.
2. **محفظة المعلم والطالب:** تطبيق (Nexus EDU) يحتوي على نظام تسجيل الدخول، الصفحة الرئيسية التفاعلية، لوحة التحكم الخاصة بالطالب، وشاشات الاختبارات التفاعلية وعروض التقدم.
3. **التواصل اللحظي والذكاء الاصطناعي:** دمج مساعد ذكاء اصطناعي (GPT) لدعم الطلاب ومشاركة شاشات تفاعلية لفصول مباشرة للتعليم المدمج.
4. **أمان وتقييمات تفاعلية وواجهة إدارة للمشتركين:** تحسين عمليات الاشتراك (Stripe)، وتأمين دخول المستخدمين، وعرض نتائجهم على شكل خطوط ورسوم بيانية متطورة.
